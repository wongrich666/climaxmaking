# Golden Opening 剧本首场改写器

这是一个本地网页工具，用来处理结构化短剧剧本：

- 上传 `.txt` 剧本
- 自动识别 `第1集 / 第2集 / 第一集`
- 每集只调用一次模型
- 只改写该集的第一个场景
- 把“黄金3秒法则 / Golden Opening”融入开篇
- 输出完整剧本并支持下载

## 支持的剧本结构

建议使用类似下面的格式：

```text
《XXX》40集完整剧本、
·故事梗概
...
·人物小传
男主：...
女主：...
·核心场景
...
·剧本正文：
第一幕：...
第一集
第1场：...
第二集
第1场：...
```

说明：

- `·故事梗概 / ·人物小传 / ·核心场景` 会原样保留
- 只会更新 `·剧本正文` 里的内容
- 每集仅替换第一个场景，其余场景保持原文

## 环境配置

1. 安装依赖

```bash
pip install -r requirements.txt
```

2. 复制配置文件

```bash
copy ..env.example ..env
```

3. 在 `.env` 中填写你要使用的模型配置

- 默认读取 `API`
- 支持 `ollama`、`deepseek`、`gemini`、`claude`

## 启动

```bash
python main.py
```

浏览器打开：

```text
http://127.0.0.1:5000
```

## 改写规则

系统会对每一集的第一个场景附加以下约束：

- 开头 1-3 句快速切入高潮或强冲突
- 强化爽点、危机、羞辱、反转、压迫感
- 允许轻微疯感
- 不能脱离原有剧情
- 保留结构化场景格式

## 当前实现范围

这是一个简化版项目，不包含原参考仓库里的多阶段快照、审核流、项目管理等能力，只保留最核心的本地上传与首场改写流程。
